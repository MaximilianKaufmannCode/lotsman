# Основная таблица реестра

Страница «Реестр» — главная рабочая поверхность Лоцмана. Здесь отображаются все активные документы по всем компаниям.

## Колонки таблицы

| Колонка | Что показывает |
|---|---|
| **Компания** | Название партнёрской компании. Колонка закреплена слева (не прокручивается). |
| **Тип** | Категория документа: договор, лицензия, аудиторский отчёт и другие. |
| **№ документа** | Регистрационный номер документа. |
| **Действ. до** | Дата истечения срока действия. Если не задана — документ без фиксированного срока. |
| **Ответственный** | Сотрудник, отвечающий за актуализацию. |
| **Статус** | Вычисляемая категория срочности (подробнее ниже). |
| **Заметки** | Произвольный комментарий. Колонка скрыта по умолчанию на узких экранах. |

Порядок и видимость колонок можно настроить через кнопку **«Колонки»** в тулбаре. Колонка «Компания» закреплена и не может быть скрыта.

## Статусы документов

Статус вычисляется автоматически из даты истечения и не хранится в базе данных. Каждый статус отображается цветом, значком и текстовой меткой одновременно — в соответствии с требованиями доступности.

| Статус | Цвет строки | Когда присваивается |
|---|---|---|
| **ОК** | зелёный | Срок истекает позже чем через 30 дней, либо срок не задан |
| **Скоро** | жёлтый | До истечения срока осталось 0–30 календарных дней (включая сегодняшнюю дату) |
| **Просрочено** | красный | Срок истёк (дата истечения раньше сегодняшней) |
| **В архиве** | серый | Документ переведён в архив независимо от даты |

Граница «Скоро» считается по календарным дням без поправки на выходные и праздники.

## Закреплённая колонка и горизонтальный скролл

Колонка «Компания» закреплена с левого края. При горизонтальной прокрутке она остаётся видимой, и вы всегда знаете, к какой компании относится строка. Граница закреплённой колонки обозначена тенью.

## Виртуализация: таблица с тысячами строк

Реестр использует виртуализацию (TanStack Virtual): в DOM-дереве браузера одновременно находятся не более 500 строк, независимо от общего числа записей. Это позволяет таблице работать плавно (60 fps) даже при 10 000+ документов. Виртуализация работает незаметно — прокрутка ведёт себя как обычно.

## Копирование значений в Excel

Выделите текст в ячейке двойным кликом (или тремя кликами для всего значения), затем нажмите **Ctrl+C** / **Cmd+C**. Значение копируется в буфер обмена и вставляется в Excel стандартным образом. Для экспорта всего реестра или отфильтрованного среза используйте кнопку **«Экспорт .xlsx»** — см. [Вложения и экспорт](attachments-and-export.md).

## Адаптивность

| Ширина экрана | Что меняется |
|---|---|
| ≥ 1280 пикс. (монитор 15"+) | Все колонки отображаются. |
| 768–1279 пикс. (ноутбук 13–14") | Колонки «Заметки» и «Ответственный» скрываются. Горизонтальный скролл. Колонка «Компания» остаётся закреплённой. |
| < 768 пикс. (телефон) | Строки отображаются карточками. |

## Навигация с клавиатуры

| Действие | Клавиша |
|---|---|
| Перейти к следующей / предыдущей строке | ↑ / ↓ |
| Перейти к следующей / предыдущей колонке | → / ← |
| Начать inline-редактирование ячейки (редактор и администратор) | Enter или двойной клик |
| Отменить редактирование и вернуть исходное значение | Esc |
| Открыть боковую панель документа | Enter на строке |
| Перейти между интерактивными элементами | Tab |

Фокус всегда обозначается видимой рамкой. Скрытые колонки пропускаются при навигации Tab.
