# Руководство администратора: компании и типы документов

Страницы администрирования — `/admin/assets` (Компании) и `/admin/document-types` (Типы документов) — открыты только роли **«Администратор»**.

Одно важное исключение: **создать компанию может и редактор** — прямо из формы добавления документа, не заходя на страницу администрирования (см. [Добавление документов](add-edit-archive.md)). Редактирование, архивирование и восстановление компаний, а также любые операции с типами документов остаются за администратором.

| Операция | Кто может |
|---|---|
| Создать компанию | редактор, администратор |
| Редактировать / архивировать / восстановить компанию | только администратор |
| Создать / изменить тип документа | только администратор |

---

## Компании (`/admin/assets`)

Компания — это партнёрская организация, к которой привязаны документы реестра. На странице **Компании** администратор ведёт справочник: создаёт, переименовывает, архивирует и восстанавливает записи.

### Создать компанию

1. Нажмите **«Добавить компанию»** на странице Компании.
2. Заполните поля:
   - **Название** — официальное наименование компании (обязательно, до 500 символов). Название должно быть уникальным среди активных компаний.
   - **ИНН** — необязательное поле. Его можно оставить полностью пустым — компания сохранится без ИНН. Если значение введено, оно проверяется:
     - 10 цифр — для юридических лиц.
     - 12 цифр — для ИП.
     - Контрольная цифра проверяется по алгоритму ФНС. Некорректный ИНН не сохранится — появится ошибка «Некорректный ИНН (10 или 12 цифр, проверочная цифра)».
   - **Заметки** — произвольный комментарий.
3. Нажмите **«Создать»**.

Одинаковый ИНН у разных компаний допускается (филиалы и холдинги могут иметь один ИНН).

> **Подсказка.** Если компания понадобилась во время заведения документа, её не обязательно создавать здесь: в форме добавления документа есть кнопка **«+ Создать компанию»**. Новая запись сразу подставится в документ. Это доступно и редактору — см. [Добавление документов](add-edit-archive.md).

### Переименовать или изменить данные компании

1. Найдите компанию в списке.
2. Нажмите иконку редактирования (карандаш) в строке компании.
3. Измените нужные поля и сохраните.

Все ранее введённые документы автоматически отображают новое название.

### Архивировать компанию

Архивирование компании — каскадная операция: все её активные документы тоже переходят в архив.

1. Нажмите иконку архивирования напротив нужной компании.
2. В диалоге подтверждения отображается количество активных документов, которые будут архивированы, например: «12 активных документов будут архивированы».
3. Нажмите **«Архивировать»**.

Документы, уже находящиеся в архиве, не затрагиваются (операция идемпотентна). Компания и все её документы исчезнут из основного вида реестра, но останутся в истории изменений.

### Восстановить компанию из архива

На странице `/admin/assets` нет элемента интерфейса для восстановления архивной компании: список не показывает архивные записи, переключателя статуса нет. Восстановление возможно только через API — администраторский запрос `PATCH /api/v1/assets/{id}/status` со статусом `active`, который снимает отметку об архивировании.

Документы при восстановлении компании **не возвращаются автоматически** — каждый нужный документ восстанавливается отдельно (см. [Восстановить архивный документ](add-edit-archive.md#восстановить-архивный-документ)).

## Типы документов (`/admin/document-types`)

Тип документа задаёт категорию (договор, лицензия и т. д.) и **глобальное расписание уведомлений** для всех документов этого типа. Создавать и изменять типы может только администратор.

### Создать тип документа

1. Нажмите **«+ Добавить тип»** на странице Типы документов.
2. Заполните поля:
   - **Код** — латинский идентификатор: строчные буквы, цифры и подчёркивание, причём начинаться он должен с латинской буквы, например `nda` или `audit_report`. Максимум 64 символа. Код нельзя изменить после создания.
   - **Отображаемое название** — название, которое видят пользователи в выпадающем списке, например «Договор» или «Сертификат соответствия». До 200 символов.
   - **Предупреждения (дней до истечения)** — через сколько дней до истечения срока отправлять напоминания. Введите несколько значений через запятую, например: `30, 14, 7, 1`. Каждое значение должно быть положительным числом.
   - **Уведомить в день истечения** — включите, чтобы система отправила уведомление непосредственно в день истечения срока.
   - **Повтор уведомления при просрочке (дней)** — как часто повторять уведомление после истечения срока. Минимум 1. Например, значение `7` означает: уведомлять каждую неделю до закрытия документа.
3. Нажмите **«Сохранить»**.

> **Подсказка.** Администратор может создать тип на лету прямо из формы документа — кнопкой **«+ Создать тип документа»**. Код типа при этом подставляется из названия автоматически (его можно поправить вручную до сохранения).

### Изменить правила уведомлений

1. Найдите нужный тип в списке и нажмите **«Редактировать»**.
2. Измените значения предварительных уведомлений, флаг «в день истечения» или интервал повторов.
3. Сохраните. Новые правила применятся ко всем документам этого типа при следующей проверке планировщика.

## Что делать, если...

- **Компания с таким названием уже существует** — активная компания с этим именем уже есть в реестре. Проверьте список перед созданием. Если старая компания в архиве, можно создать новую с тем же именем.
- **ИНН не принимается** — ошибка возникает только при введённом некорректном ИНН: проверьте длину (10 или 12 цифр), что введены только цифры, и контрольную цифру. Если ИНН неизвестен, просто оставьте поле пустым — это норма, компания сохранится без ИНН.
