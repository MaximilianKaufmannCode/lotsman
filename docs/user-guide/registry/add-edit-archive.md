# Добавление, редактирование и архивирование документов

Для создания и редактирования документов необходима роль **редактора** или **администратора**. Читатели могут только просматривать.

## Добавить документ

1. Нажмите **«+ Добавить документ»** в правом верхнем углу тулбара.
2. Заполните форму. Обязательные поля:
   - **Компания** — выберите из списка (создаёт только администратор; см. [Руководство администратора](admin.md)).
   - **Тип документа** — выберите из списка (например, «Договор», «Лицензия»).
   - **№ документа** — регистрационный номер (до 500 символов).
   - **Действ. до** — дата истечения. Если срок не ограничен, оставьте пустым; статус будет «ОК».
3. Необязательные поля: **Дата выдачи**, **Ответственный**, **Заметки** (до 10 000 символов).
4. Нажмите **«Сохранить»**. Новый документ появится в таблице без перезагрузки страницы.

После создания система автоматически планирует уведомления о сроках на основании настроек типа документа.

## Редактировать ячейку напрямую (inline-редактирование)

Дважды кликните на любую ячейку таблицы — она перейдёт в режим редактирования. Поле заполнится текущим значением.

- **Enter** — сохранить изменение.
- **Esc** — отменить и вернуть исходное значение.

Сохранение подтверждается уведомлением «Изменения сохранены». При сетевой ошибке появляется «Не удалось сохранить изменения. Попробуйте снова» — ячейка возвращается к последнему серверному значению.

Если другой пользователь изменил этот же документ пока вы его редактировали, после сохранения появится уведомление «Документ был изменён другим пользователем». Нажмите **«Обновить»**, чтобы загрузить актуальные данные.

## Боковая панель документа (side drawer)

Одиночный клик на строку открывает боковую панель со всеми полями документа. Панель содержит три вкладки:

| Вкладка | Содержимое |
|---|---|
| **Основное** | Все поля документа, статус, ответственный. |
| **Вложения** | Список прикреплённых файлов; кнопка «Добавить файл» (для редакторов и администраторов). |
| **История изменений** | Хронологический список изменений: кто, когда, какое поле, старое и новое значение. |

Если документ находится в архиве, в верхней части панели отображается баннер «Документ в архиве», а все поля редактирования отключены. Кнопка «Восстановить» видна только администратору.

## Архивировать документ (мягкое удаление)

Архивирование убирает документ из основного вида реестра, но не удаляет его. Документ сохраняется и доступен для аудита.

**Один документ:**
1. Откройте боковую панель (одиночный клик на строку).
2. Нажмите **«Архивировать»**.
3. Подтвердите в диалоге.

**Несколько документов (bulk-архивирование):**
1. Отметьте нужные строки галочками слева (максимум 100 за раз).
2. В тулбаре появится кнопка **«Архивировать N»** — нажмите её.
3. Подтвердите в диалоге «Вы уверены? N документов будут архивированы».

Если выбрано больше 100 строк, 101-я галочка не устанавливается и появляется предупреждение «Максимум 100 документов за одну операцию». Разбейте операцию на несколько частей.

Читатели не видят галочки выбора строк.

## Восстановить архивный документ

Восстановление доступно только **администратору**.

1. Найдите архивный документ: включите фильтр «Статус = В архиве» или воспользуйтесь поиском.
2. Откройте боковую панель — нажмите **«Восстановить»**.
3. Документ снова появится в основном реестре со статусом «ОК» (или «Скоро» / «Просрочено» в зависимости от даты).
